** Some fixes i needed to apply once
- fix for when adding tailscale0 interface to unmanaged devices in
  networkmanaged makes discord clients infinitely 'DTLS Connecting':
  - launch discord client via ~firejail --net=wlan0 --noprofile <client>~, replace
    wlan0 with your active network interface ([[https://github.com/tailscale/tailscale/issues/10396#issuecomment-3871203280][source]])
- tailscale access local addresses using the homelab:
  - [[https://tailscale.com/docs/features/subnet-routers#connect-to-tailscale-as-a-subnet-router][main doc]]:
    - host machine:
      - enable IP forwarding (1st option that is there)
      - advertise subnet routes
        - sudo tailscale set --advertise-routes=192.168.1.1/32
          - 32 is for specific ip, if needing to advertise whole local network use 192.168.1.0/24
    - tailscale admin panel:
      - add ACL as shown in main doc, replace example IP ranges with yours
        #+begin_src
       {
	  "src": ["seme4eg@github", "192.168.1.1/32"],
	  "dst": ["tag:main", "192.168.1.1/32"],
	  "ip":  ["*"],
       }
        #+end_src
    - client machine (IMPORTANT):
      - ~sudo tailscale set --accept-routes~ on to discover
        newly advertised IPs
    - ...
    - remove IP forwarding & disadvertise the stuff
- fix for hyprland being sluggish & unresponsive after sleep & loading cpu by
  4.4% constantly:
  1. Don't unplug the egpu, _first power it off_, then after some time unplug
  2. read [[https://wiki.hypr.land/Configuring/Performance/#:~:text=much%20about%20animations.-,Fractional%20scaling,updated%20on%20March%2027%2C%202026][Performance hypr wiki page]]
  3. +change display scale from 1.666667 to 1.6+
  4. its prob some power saving mode fuckup on amd side, read =amd-udev.sh= file
  5. Fix Intel iGPU Stutter: If using Intel, TLP defaults can be too aggressive. Edit /etc/tlp.conf and set INTEL_GPU_MIN_FREQ_ON_AC to a higher value (e.g., 500).
